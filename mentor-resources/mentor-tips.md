# Mentor Tips & Strategies

A grab-bag of things that work.

## Building Confidence

- Celebrate small wins — first commit, first push, first PR.
- Normalize mistakes — "I make merge conflicts too."
- Show your own GitHub profile — including the messy commits.
- Encourage peer learning — let mentees teach mentees.
- Use mentee names in examples ("`feat: ada-loves-hamilton`").

## Constructive Feedback

- ✅ Specific: "Your commit message is clear because it explains the *why*."
- ✅ Actionable: "Try adding more detail to explain the trade-off."
- ✅ Growth-focused: "Next time, try resolving this without `git reset --hard`."

## Red Flags 🚩

- 🚩 Mentee skips projects (just attends)
- 🚩 Consistently incomplete work
- 🚩 Not asking questions
- 🚩 Anxiety about "breaking" Git

When you see these, schedule a 1:1, listen first, and reduce scope before adding more.

## Energy Management

- 90 minutes is the absolute max for a single session.
- Take a 5-minute break at minute 50 — every time.
- End with a clear "what's due" — fuzzy endings tank momentum.

## Things to Steal

- A shared `kudos.md` file mentees can PR into ("Today X helped Y debug").
- Friday "PR parade" — 10 min showing off what got merged this week.
- A ban on `git push --force` until Week 6.
