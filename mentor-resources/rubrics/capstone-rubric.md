# Capstone Rubric (generic — applies to all weeks)

Total: 100 points. Pass: ≥ 70.

| Criterion | Points | What "great" looks like |
| --------- | -----: | ----------------------- |
| **Functionality** | 30 | Project does what the brief asked. No crashes. |
| **Git hygiene** | 25 | Atomic commits, conventional messages, clean history. |
| **README quality** | 15 | Clear description, install steps, screenshots. |
| **Code/file organization** | 10 | Sensible folder structure, no junk files. |
| **`.gitignore` correctness** | 5 | No secrets, no `node_modules/`, no `.DS_Store`. |
| **Reflection / "what I learned"** | 10 | Honest, specific, names a stuck moment. |
| **Stretch goal** | 5 | Anything above and beyond the brief. |

## Feedback rules

- Always lead with one specific thing they did well.
- Limit to **3 improvement asks**, ranked.
- End with the next concrete action.
- Turnaround target: 48 hours.

## Quick grading checklist

- [ ] Repo clones cleanly?
- [ ] README has install + run instructions?
- [ ] No secrets committed?
- [ ] Commits readable in `git log --oneline`?
- [ ] At least one branch + PR (if week's curriculum required it)?
