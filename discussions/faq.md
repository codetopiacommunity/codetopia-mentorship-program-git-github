# FAQ

### Do I need to know how to code already?

No. The program assumes only basic comfort with the terminal. We'll build everything else.

### Mac, Windows, or Linux?

All three work. Windows users: install [Git for Windows](https://gitforwindows.org/) and use Git Bash for the terminal.

### How long does each session take?

60–90 minutes of focused time, plus 30–60 minutes of project work.

### What if I miss a session?

Read the `lecture-notes.md` for that session, do the project, and ask any questions in Discussions.

### What if I break my repo?

You almost certainly haven't. See [`../troubleshooting/lost-commits.md`](../troubleshooting/lost-commits.md). Worst case: re-clone.

### Can I use GitHub Desktop / GitKraken / VS Code's Git UI?

Yes — but for the first 4 weeks, please use the command line so the model in your head matches what's happening. After Week 4, GUIs are fair game.

### Do I need a paid GitHub account?

No. Free accounts give you everything we use.

### What's the difference between Git and GitHub?

**Git** is the version control tool that runs on your computer. **GitHub** is a website that hosts Git repositories and adds collaboration features (PRs, issues, reviews).

### How do I get my capstone reviewed?

Open a PR per [CONTRIBUTING.md](../CONTRIBUTING.md). Tag your mentor.

### Can I share my mentee work publicly?

Yes — that's the goal. Your `projects-showcase/` entry is portfolio fuel.
