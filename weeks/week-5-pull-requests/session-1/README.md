# Week 5 · Session 1 · Forks and Pull Requests

> **XP:** 75
> **Time:** 60–90 min
> **Prereqs:** Weeks 1–4. You can branch, push, and keep a remote in sync.

## Learning Objectives

By the end of this session you will:

1. Explain the difference between a **branch** and a **fork**.
2. Fork a GitHub repo and clone your fork locally.
3. Set up an `upstream` remote so you can pull changes from the original.
4. Push a feature branch and open a pull request.
5. Write a PR description that a reviewer can act on in under 60 seconds.
6. Request a review from a specific person.
