## Big Picture

A pull request is not a Git feature. It's a **conversation** wrapped around a branch. This week teaches:

1. How to set up the *collaboration shape* (fork vs branch, upstream vs origin).
2. How to propose work clearly (PR title, description, screenshots, linked issues).
3. How to review work kindly and specifically (suggested changes, approvals, request-changes).
4. How to close the loop (addressing feedback, choosing merge commit vs squash vs rebase).

By end of week, mentees should comfortably author and review a PR without a mentor hovering.
