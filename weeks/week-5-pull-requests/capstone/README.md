# Week 5 Capstone — "PR Against the Real Repo"

> **XP:** 250 base + up to 50 stretch

## Brief

This is the first capstone where you ship *against this actual mentorship repo*. You'll:

1. Fork this repo.
2. On a feature branch, add a Week 5 showcase entry under `projects-showcase/week-5-projects/<your-handle>/`.
3. Open a well-described PR.
4. **Review at least one peer's PR** with at least three substantive comments, one of which uses the suggested-changes feature.

The bar is "a PR I'd happily merge at work" — both as author and as reviewer.

## Requirements

### As author

- Fork created; `upstream` remote configured on your clone.
- Feature branch named in conventional format (e.g., `feat/add-<handle>-week-5-showcase`).
- At least **3 commits** on the branch, all conventionally formatted.
- PR opened against this repo's `main` branch (not your fork's `main`).
- PR description filled out using [`../../../templates/pr-template.md`](../../../templates/pr-template.md) — no `<!-- comment -->` leftovers, no empty sections.
- Your mentor requested as reviewer.
- A folder at `projects-showcase/week-5-projects/<your-handle>/` containing:
  - `README.md` — a short writeup of your Week 4 project with screenshots and lessons.
  - Any assets you want to link from it.
- All review comments either addressed with a new commit or replied to in the thread.

### As reviewer

- Review at least **one** peer's Week 5 PR.
- Leave at least **3 substantive comments**, including:
  - At least one **suggested change** (triple-backtick `suggestion` block).
  - At least one **question:** or **praise:** prefixed comment.
- Submit with an explicit verdict (Comment / Approve / Request changes) — not a bare save.
- If you pick Request changes, clearly name the blocker(s) in the review summary.

## Deliverable

Your PR itself is the deliverable. Mentors will grade:

- The PR's diff and description.
- The PR's commit history.
- Your review activity on a peer's PR (they'll check the Conversation tab).

No separate discussion post required — the PR is the portfolio piece.

## Grading

See [`rubric.md`](rubric.md). Pass = 70+. Use [`checklist.md`](checklist.md) before submitting.

## Stretch (+50 XP)

| Item | XP |
| ---- | -: |
| Your PR earned a real-word merge from a mentor using **squash** with a conventional squashed commit message | +15 |
| You reviewed **two** peer PRs (not just one) with full review quality on both | +15 |
| At least one of your suggested-changes was **accepted and committed** by the other author | +10 |
| Your PR resolves cleanly without any force-push during review | +10 |
