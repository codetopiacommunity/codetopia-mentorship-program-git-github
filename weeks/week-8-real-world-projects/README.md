# Week 8 — Real-World Projects

> **XP:** 450 (150 + 150 + 150 capstone) — **the graduation week**

You've learned Git. You've learned GitHub. You've shipped a PR to a stranger's code. This week is the finale: we take what you've built and make it *real*. Automation, releases, a README that looks like a professional project.

By the end, you will have a repo you're proud to put at the top of your résumé.

## Theme

**Shipping.** Everything in the final week is about the last 10%: the CI that proves your code works, the release tag that marks a moment, the README that tells a stranger why they should care.

## Capstone — Program Graduation

[**The final project.**](./capstone/README.md) Take any earlier project (or a new one), add CI via GitHub Actions, cut a `v1.0.0` release, and polish the README until it looks portfolio-grade.

This is the one you link on LinkedIn.

## Learning objectives

By the end of the week, you should be able to:

- Write a GitHub Actions workflow from scratch that runs on push and PR.
- Add a lint gate and a test gate that block merges on failure.
- Explain semantic versioning and apply it.
- Create an annotated tag and promote it to a GitHub Release with release notes.
- Write a README that hits the professional quality bar.
- Host a static site for free with GitHub Pages.

## The mindset this week

> **Strangers will evaluate your repo in 30 seconds.** The README, the badges, the structure — those are the 30 seconds. CI and releases are the evidence that you're serious.
