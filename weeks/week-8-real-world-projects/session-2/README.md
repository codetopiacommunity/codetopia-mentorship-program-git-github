# Session 2 — Releases, Tags, and a Portfolio-Ready README

> **XP:** 75 (attendance + exercises)

## Overview

Your code works. CI is green. Now you announce it.

This session we mark a moment in your project's history (`v1.0.0`), turn that mark into a published release with release notes, and polish the README so that a recruiter who stumbles on your repo understands — in 30 seconds — what it is and why they should care.

Optionally: host a live demo with GitHub Pages, for free, in 5 minutes.

## Objectives

- Apply semantic versioning (SemVer) correctly.
- Create an annotated tag with `git tag -a` and push it.
- Promote a tag to a GitHub Release with release notes.
- Write a README that hits the professional quality bar.
- Deploy a static site or docs with GitHub Pages.

## Mastery checks

- [ ] I can explain the rules for bumping major/minor/patch.
- [ ] I can create an annotated tag vs a lightweight tag, and say which to use when.
- [ ] I can list the 6 things every portfolio README needs.
- [ ] I can deploy a simple site to GitHub Pages.
