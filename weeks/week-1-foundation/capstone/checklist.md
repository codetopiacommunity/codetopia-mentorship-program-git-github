# Week 1 Capstone — Pre-submission Checklist

Run this list before opening your PR.

## Repo

- [ ] `git status` returns "nothing to commit, working tree clean"
- [ ] `git log --oneline` shows ≥ 8 commits
- [ ] Every commit message starts with a type (`feat:`, `fix:`, `docs:`, …)
- [ ] No commit has a past-tense verb ("added", "fixed")
- [ ] No commit message ends with a period

## Files

- [ ] `README.md` present and complete
- [ ] `.gitignore` present
- [ ] `screenshots/` folder present with at least 2 named screenshots (`01-...`, `02-...`)
- [ ] Best screenshot shared in the WhatsApp group
- [ ] No `.DS_Store`, `Thumbs.db`, or `.env` committed
- [ ] No accidental personal info (real address, phone, secret tokens)

## Content

- [ ] An "About Me" file exists (HTML or `about.md`)
- [ ] At least 3 sections of real content
- [ ] "What I Learned This Week" section in README

## Submission

- [ ] You forked the program repo
- [ ] You added your folder under `projects-showcase/week-1-projects/<handle>/`
- [ ] PR title is descriptive
- [ ] You tagged your mentor
