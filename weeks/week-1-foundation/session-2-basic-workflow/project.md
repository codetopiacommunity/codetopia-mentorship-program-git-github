# Project — Session 2

> **XP:** 50

## Build: a personal `my-favorites` repo

Practice the add → commit cycle by building a tiny markdown "favorites" notebook from scratch.

## Steps

1. `mkdir my-favorites && cd my-favorites`
2. `git init`
3. Create a `README.md` with a header.
4. Make **at least 5 commits**, each adding *one* small thing:
   - Favorite books
   - Favorite movies
   - Favorite music
   - Favorite quotes
   - A "credits" footer
5. Each commit message must follow the conventional format.

## Deliverable

Submit your work as follows:

1. **In your project's `screenshots/` folder** (the submission of record): add at least one screenshot — typically a clean `git log --oneline` and the rendered `README.md`. Name them `01-git-log.png`, `02-readme.png`, etc.
2. **In the WhatsApp group** (informal share): drop a copy of your best screenshot so the cohort can cheer you on.
3. **In Discussions** (additional channel): paste the text of `git log --oneline` and a snippet of your `README.md`.

See [CONTRIBUTING.md → Screenshot conventions](../../../CONTRIBUTING.md#screenshot-conventions).

## Stretch (+25 XP)

- Make one commit that fixes a typo (use `fix:`).
- Make one `docs:` commit that adds a "How to read" section.
