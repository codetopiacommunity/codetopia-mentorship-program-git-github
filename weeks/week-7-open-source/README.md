# Week 7 — Open Source Contribution

> **XP:** 400 (150 + 150 + 100 capstone)

This week you step outside your own repos and contribute to someone else's code. It's the first time in the program you'll be interacting with people you don't know, on a project you don't control. It's also one of the most rewarding things you can do as a developer.

## Theme

Open source runs on goodwill. Everything you'll learn this week — finding issues, reading CONTRIBUTING.md, the etiquette of "is this still being worked on?" — is about being a good guest in someone else's house.

## Capstone

[**Your first (or simulated) open-source PR.**](./capstone/README.md) Identify a real project, pick an issue, make the change, and open a PR — or write a thorough simulated PR if nothing's ready to ship. Either way you'll learn the full loop.

## Learning objectives

By the end of the week, you should be able to:

- Find "good first issues" on GitHub effectively.
- Evaluate whether a project is healthy and accepting contributions.
- Read and follow a `CONTRIBUTING.md` without being told.
- Write a PR description that a maintainer will actually want to merge.
- Respond to maintainer feedback without taking it personally.
- Explain the unwritten social contract of open source.

## The lens for this week

> **Your PR is a gift, not a demand.** Maintainers owe you nothing. Your job is to make their life easier, not harder.

Hold that thought through every decision you make this week.
