# Session 1 — Finding Good First Issues

> **XP:** 75 (attendance + exercises)

## Overview

Finding the right first issue is half the battle. A well-chosen issue in a healthy project leads to a merged PR in days. A poorly chosen issue in a dead project leads to weeks of waiting and frustration.

This session is about *triage* — evaluating projects and issues before you invest any effort.

## Objectives

- Use GitHub's search to find "good first issues" you can actually do.
- Evaluate project health (activity, responsiveness, CI, CONTRIBUTING.md).
- Read CONTRIBUTING.md, CODE_OF_CONDUCT.md, and issue templates critically.
- Practice the etiquette of "is this still being worked on?" without sounding like a bot.

## Mastery checks

- [ ] I can craft a GitHub search query that returns issues matching my skill set.
- [ ] Given a project URL, I can list 5 signals that tell me whether it's healthy.
- [ ] I can identify 3 red flags that suggest an issue isn't a good first PR.
- [ ] I can write a polite, non-presumptuous "I'd like to work on this" comment.
