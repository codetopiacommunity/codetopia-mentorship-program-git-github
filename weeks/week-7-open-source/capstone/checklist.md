# Week 7 Capstone — Pre-Submission Checklist

## Target

- [ ] I triaged the project's health using the three-lens approach.
- [ ] CONTRIBUTING.md was read, not skimmed.
- [ ] The issue is unclaimed OR I got explicit confirmation I can take it.
- [ ] The change is one logical thing.

## Branch + commits

- [ ] Branch named per target's convention (or sensible default like `fix/<desc>`).
- [ ] Commit messages follow target's style.
- [ ] DCO sign-off if required (`-s` flag used).
- [ ] No unrelated changes in the diff.

## PR description

- [ ] **What** — clear, specific.
- [ ] **Why** — motivation, not just symptoms.
- [ ] **How** — right level of abstraction.
- [ ] **Testing** — concrete steps and results.
- [ ] Issue linked if relevant (`Closes #NNNN`).
- [ ] Screenshots for UI changes.
- [ ] Any project-mandated checklist items checked.

## Writeup

- [ ] `WRITEUP.md` covers: how I found it, how I triaged, what I did, what happened.
- [ ] Maintainer interactions (or simulated ones) are quoted with context.
- [ ] Reflection is specific, not generic.
- [ ] Path A: real PR URL included.
- [ ] Path B: `PR-DESCRIPTION.md` and `SIMULATED-REVIEW.md` included.

## Etiquette

- [ ] I was polite in every interaction.
- [ ] I thanked the maintainer (if I got a response).
- [ ] I did not tag personal GitHub accounts outside the project.
- [ ] I did not ping maintainers more than once.

## Cohort submission

- [ ] Folder under `projects-showcase/week-7-projects/<your-handle>/`.
- [ ] PR title is Conventional Commits.
- [ ] PR description follows [`../../../templates/pr-template.md`](../../../templates/pr-template.md).
- [ ] I checked off this list in my PR description.

## Final sanity check

- [ ] If a maintainer from the target project read my writeup, I would be comfortable with what they saw.
- [ ] My PR (or simulated PR) is a *gift*, not a *demand*.

If every box is checked, you're ready.
